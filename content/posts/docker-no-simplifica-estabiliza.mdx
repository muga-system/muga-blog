El cambio principal no fue tecnico, fue conceptual.

Docker no se usa para facilitar, se usa para evitar variabilidad.

## Implementacion

Cada servicio paso a vivir en su contenedor.

## Hallazgo de implementacion

Con contenedores, las diferencias de maquina dejaron de afectar el comportamiento.

## Resultado del trabajo diario

El sistema dejo de depender de la maquina.
Paso a depender de su definicion.

## Conclusión operativa

Docker no simplifica el trabajo: estabiliza el contexto.
