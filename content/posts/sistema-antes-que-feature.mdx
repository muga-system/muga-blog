date: "2026-04-05"
description: "Evitar construir features aisladas permitio mantener coherencia estructural."
area: "decisiones-arquitectura"
---
Durante el desarrollo aparecieron varias funcionalidades posibles.

No se implementaron de inmediato.

## Criterio previo

Primero se valido:

- si encajaban en el sistema
- si respetaban la arquitectura

## Hallazgo de implementacion

Validar el encaje primero evito construir piezas que despues habia que descartar.

## Resultado del trabajo diario

Menos codigo descartado.
Mas coherencia.

## Idea central

La feature correcta aparece despues de entender el sistema.
